## Findings considered and rejected

- **Daemon plaintext password in setup_token.json**: This is the design trade-off for the one-time token flow (5-minute TTL). The token file is written by `setup_wizard.py` with the plaintext password so the redeem endpoint can return it. The TTL limits exposure. In v2, consider encrypting the password with the token as key.
- **Multi-user management API**: Out of scope for v1. The register endpoint creates ONLY the first user. Future: admin-only user management.
- **Daemon systemd unit deployment**: Out of scope for this audit. The daemon runs but is not installed as a systemd unit. Deferred to a separate deployment task.
- **Streaming markdown rendering**: Explicitly deferred. Streaming text uses plain `Text()` with cursor. Markdown rendering only for finalized messages.
- **Kanban parity Phase 2 (comments, attachments, dependencies, runs, events)**: Documented in KANBAN_PARITY_SPEC.md but beyond Phase 1 scope. Create follow-up plans after Phase 1 lands.
